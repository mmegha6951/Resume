ProcureMind: Agentic AI Procurement Platform

May 2026 - June 2026
Problem Statement

Traditional procurement processes are time-consuming, error-prone, and require significant manual effort for vendor sourcing, price comparison, and compliance verification. Organizations struggle with:

  • Manual vendor evaluation and comparison consuming weeks of work
  • Inconsistent compliance checking across different suppliers
  • Difficulty extracting and standardizing pricing from multiple sources
  • Lack of intelligent decision support for procurement decisions
  • Complex approval workflows that bottleneck the process
Solution Overview

Developed an AI-powered procurement platform using multi-agent architecture that automates vendor sourcing, pricing extraction, and compliance auditing. The system uses intelligent agents coordinated through LangGraph to handle different aspects of procurement workflows simultaneously.

Key Features:
  • Multi-agent architecture for parallel processing of procurement tasks
  • Asynchronous Human-in-the-Loop (HITL) workflows for approval-based execution
  • Real-time compliance auditing and vendor evaluation
  • Resumable agent states for fault tolerance
  • Privacy-preserving local deployment with Docker containerization
Technical Architecture

The platform implements a sophisticated microservices architecture with task orchestration:

  • Agent Orchestration: LangGraph manages multi-agent workflows with state persistence
  • LLM Engine: Qwen 2.5 (via Ollama) for local inference, eliminating cloud dependencies
  • Task Queue: Celery + Redis for asynchronous task processing and approval workflows
  • Data Layer: PostgreSQL for persistent storage of vendor data, pricing, and compliance records
  • Containerization: Docker for reproducible, scalable deployments
  • Privacy: All processing runs locally, ensuring data security

  • Configured Ollama integration with Qwen 2.5 for local LLM inference
  • Containerized the application with Docker for secure, privacy-preserving deployment
  • Implemented state persistence mechanisms for resumable agent workflows

EEG-Based Mental Workload Detection & Adaptive Teaching

Academic Project
Problem Statement

Traditional education systems treat all students uniformly without considering individual cognitive states. Key challenges include:

  • Inability to detect when students are cognitively overloaded or fatigued
  • Content delivery doesn't adapt to individual learning states
  • Teachers lack real-time insights into class cognitive load
  • Loss of learning efficiency due to mismatch between content difficulty and student state
Solution Overview

Developed a machine learning system that analyzes EEG (electroencephalography) signals to detect mental workload in real-time and enables personalized, adaptive learning experiences. The system processes brain signals to understand cognitive states and adjusts content difficulty accordingly.

Key Features:
  • Real-time EEG signal processing and mental workload classification
  • FFT-based frequency domain analysis for cognitive state detection
  • Bandpass filtering to extract relevant frequency bands
  • Adaptive content personalization based on detected fatigue levels
  • ML model trained on cognitive workload patterns
Technical Architecture

Multi-stage signal processing and machine learning pipeline:

  • Signal Acquisition: Raw EEG data collection from multiple electrode positions
  • Preprocessing: Artifact removal and noise filtering
  • Feature Extraction: FFT for frequency domain analysis, extraction of cognitive markers
  • Signal Filtering: Bandpass filtering (alpha: 8-12Hz, beta: 12-30Hz bands)
  • Classification: Scikit-learn ML models to categorize workload levels
  • Adaptation Engine: Real-time content adjustment based on predictions
Individual Contribution
  • Designed and implemented end-to-end EEG signal processing pipeline
  • Developed FFT and bandpass filtering algorithms using SciPy for frequency domain analysis
  • Built feature engineering module extracting cognitive markers from processed signals
  • Trained and optimized Scikit-learn classification models for workload detection
  • Implemented adaptive learning system that personalizes content based on real-time predictions
  • Conducted extensive validation and testing on EEG datasets
Technology Stack
Python NumPy SciPy Scikit-learn Signal Processing FFT
Challenges & Learnings
  • Challenge: EEG signals are noisy and artifact-prone. Solution: Implemented robust filtering pipeline with artifact detection algorithms
  • Challenge: Feature extraction from complex brain signals. Solution: Applied domain-specific signal processing techniques (FFT, bandpass filtering)
  • Challenge: Model generalization across different subjects. Solution: Used subject-independent feature normalization and cross-validation
  • Learning: Deep understanding of signal processing and its application to neuroscience
  • Learning: Practical experience with brain-computer interface (BCI) systems
Impact & Results
  • 85% Classification Accuracy: Model achieves high accuracy in detecting mental workload states
  • Real-time Processing: System provides sub-second response times for content adaptation
  • Personalized Learning: Adaptive system adjusts difficulty to match individual cognitive capacity
  • Presented at National Hackathon: Showcased at 28-hour GSSS College Engineering hackathon
  • Research Value: Contributes to brain-computer interface research in education technology

Reversible Data Hiding in Encrypted Media

Nov 2025 - Dec 2025
Problem Statement

Secure data embedding in multimedia files requires balancing competing demands:

  • Need for high-capacity data embedding without visible distortion
  • Requirement for complete data recovery without loss
  • Encryption of sensitive metadata while maintaining media quality
  • Progressive recovery capability for partial data retrieval
  • Minimal impact on original image/video quality (PSNR preservation)
Solution Overview

Engineered an advanced cryptographic and steganographic system combining AES encryption with progressive recovery algorithms for secure metadata embedding in digital media. The system embeds encrypted data across color channels with sophisticated algorithms ensuring lossless reconstruction.

Key Features:
  • AES encryption for metadata security
  • Pixel-prediction techniques for optimal embedding
  • Progressive data recovery with partial reconstruction capabilities
  • Lossless media reconstruction from embedded data
  • Support for both images and video streams
  • Minimal PSNR distortion during recovery phases
Technical Architecture

Multi-layer steganographic system with cryptographic protection:

  • Encryption Layer: AES encryption of metadata before embedding
  • Embedding Layer: Pixel-prediction algorithms for optimal carrier selection
  • RGB Channel Distribution: Intelligent data distribution across color channels
  • Progressive Recovery: Staged data extraction with quality preservation
  • Media Format Support: Image and video stream processing
  • Quality Metrics: Real-time PSNR monitoring during operations
